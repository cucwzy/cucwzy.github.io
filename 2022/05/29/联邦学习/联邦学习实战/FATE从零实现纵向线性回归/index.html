FATE从零实现纵向线性回归

2022年05月29日

一、制作数据集

  • 数据集：波士顿房价预测数据集，样本数506，13个特征，标签是房屋得均价
  • 切分数据集：将前406条数据作为训练数据，后100条数据作为测试数据
    • 训练数据集切分：随机抽取360条数据和前8个特征作为机构A的本地数据，随机抽取380条数据和后5个特征以及标签作为机构B的本地数据
    • 测试数据集切分：随机抽取80条数据和前8个特征作为机构A的本地测试数据，随机抽取85条数据和后5个特征以及标签作为机构B的本地测试数据

二、通过DSL Conf运行训练和预测任务

2.1 数据输入

  • upload_train_host_conf.json
  • upload_train_guest_conf.json
  • upload_eval_host_conf.json
  • upload_eval_guest_conf.json
  • 上传数据命令

2.2 模型训练

2.2.1 配置DSL文件

  • 与横向联邦学习相比较，纵向联邦学习需要进行样本对齐，即在不泄露双方数据的前提下，求取出双方用户的交集，从而确定模型训练的训练数据集。
  • 关于组件的说明
    • reader_0: 数据读取组件（v2版本加入），支持图像数据
    • dataio_0: 数据 I/O 组件
    • intersection_0: 样本对齐组件
    • hetero_linr_0: 纵向线性回归模型组件
    • evaluation_0: 模型评估组件
  • 官方提供的示例可以在以下目录找到
    • /examples/dsl/v1/hetero_linear_regression/test_hetero_linr_train_job_dsl.json，直接可以使用
  • /examples/dsl/v2/hetero_linear_regression/test_hetero_linr_train_job_dsl.json，需要修改，增加evaluation_0组件。（相比v1版本，v2版本增加了reader组件，支持图像输入）

2.2.2 运行配置Submit Runtime Conf

  • /examples/dsl/v1/hetero_linear_regression/test_hetero_linr_train_job_conf.json需要修改party ID为对应ID
    • 指定数据对应上传数据时的设置
    • 修改label_name
    • 设置模型参数
  • /examples/dsl/v2/hetero_linear_regression/test_hetero_linr_train_job_conf.json
    • 需要修改party ID为对应ID
    • 指定数据对应上传数据时的设置
    • 修改label_name
    • 设置模型参数
    • 设置evaluation_0相关参数

2.2.3 V1提交任务，训练模型

  • 执行pipeline任务 flow job submit -c ${conf_path} -d ${dsl_path}
  • 对比v1版本与v2版本的DAG（有向无环图）
  • v1

  • 在guest查看模型在训练数据上的效果

  • 在arbiter节点中查看训练过程中loss的变化

2.2.4 V2提交任务，训练模型

  • v2，相比v1，只多了reader组件

  • 在guest查看模型在训练数据上的效果

  • 在arbiter节点中查看训练过程中loss的变化

2.3 模型评估

2.3.1 修改DSL

  • /examples/dsl/v1/hetero_linear_regression/test_hetero_linr_validate_job_dsl.json
    • 无需修改
  • /examples/dsl/v2/hetero_linear_regression/test_hetero_linr_validate_job_dsl.json
    • 增加evaluation_0组件

2.3.2 修改conf

  • /examples/dsl/v1/hetero_linear_regression/test_hetero_linr_validate_job_conf.json
    • 修改party ID
    • 修改数据源
    • 修改label_name
    • 设置模型参数
  • /examples/dsl/v2/hetero_linear_regression/test_hetero_linr_validate_job_conf.json，需要修改
    • 修改party ID
    • 修改数据源，修改label_name
    • 设置evaluation_0组件的参数
    • 设置模型参数
